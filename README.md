# BoltFFI Ruby target — generated demo

This repo is the **output** of [boltffi/boltffi#518](https://github.com/boltffi/boltffi/pull/518)
(the Ruby binding target), so you can see exactly what the generator produces
without checking out the PR.

Everything under `lib/` and `ext/` was generated by:

```
boltffi generate ruby --experimental --output .
```

run against the annotated Rust crate in [`rust/`](rust/) (a copy of BoltFFI's
`examples/demo`). That's the input; `lib/` and `ext/` are the output. Nothing
here is hand-written except the `demo_*.rb` scripts that exercise the bindings
and this README.

## The input (`rust/`)

The annotated Rust crate that produced these bindings. The `#[boltffi(...)]`
annotations on the functions, records, enums, and `impl` blocks in `rust/src/`
are what the generator reads. Build it as a staticlib (below) to link into the
extension.

## What's generated

| File | What it is |
|------|------------|
| `lib/demo.rb` | Ruby wrapper — records as value classes, enums as constant modules, `module_function` shims over the C extension |
| `ext/demo/_native.c` | The C extension — `extern` decls for the Rust symbols, scalar conversions via Ruby's own `NUM2*` macros, `TypedData` glue for opaque classes |
| `ext/demo/extconf.rb` | Build script — links a prebuilt `libdemo_ffi.a` if present, else injects a `make`-time rule that builds the Rust staticlib from source with `cargo rustc`; hides every symbol except `Init_demo_native` |
| `demo.gemspec` | Gem spec with an explicit (non-globbed) file manifest |

The key design point: the Rust staticlib is **linked directly into a single
`demo_native.bundle`** — no `dlopen`, no runtime registry, no library search at
load time. A single `require` loads everything.

## Building and running it

You need a Rust toolchain and Ruby (>= 3.1).

`extconf.rb` injects a `make`-time rule that builds the Rust staticlib from
source — `make` runs `cargo rustc` and discovers the produced `.a` straight from
cargo's JSON output, so there's no manual `cargo` + `cp` dance. Building under
`make` (not at configure time) keeps cargo on the same footing as a normal C
compile: it honors `$CARGO`/`$RUSTFLAGS` and participates in `make clean`. It
locates the crate via `$BOLTFFI_CRATE_MANIFEST`.

The catch for *this* repo: the crate in `rust/` depends on the BoltFFI workspace
by relative path (`../../boltffi`), so it isn't standalone-buildable here. Point
`BOLTFFI_CRATE_MANIFEST` at `examples/demo` in a BoltFFI checkout (whose workspace
deps resolve). A real BoltFFI-generated gem depends on `boltffi` from crates.io and
needs no such override.

```sh
# rust/ is a copy of examples/demo; build against the real crate in a checkout
git clone https://github.com/boltffi/boltffi

# Build the extension — `make` compiles the staticlib from source, then links it
cd ext/demo
BOLTFFI_CRATE_MANIFEST=/path/to/boltffi/examples/demo/Cargo.toml ruby extconf.rb && make
cp demo_native.bundle ../../lib/    # so `require_relative "demo_native"` resolves

# Run the demos
cd ../.. && ruby run_all.rb
```

Expected tail:

```
ALL DEMO SLICES PASSED ✅  (demo_functions.rb, demo_enums.rb, demo_records.rb, demo_classes.rb, demo_ractor.rb)
```

`nm -gU lib/demo_native.bundle | grep -i init` shows only `_Init_demo_native`
exported — the entire Rust runtime is hidden.

## What the demos cover

- **`demo_functions.rb`** — scalar (i32/i64/u32/u64/f32/f64), bool, string, bytes free functions
- **`demo_enums.rb`** — enums with default, explicit, and negative discriminants
- **`demo_records.rb`** — record value classes; error records subclass `StandardError`
- **`demo_classes.rb`** — opaque `TypedData` classes: constructors, scalar/string/void/handle `&self` methods, GC-driven free (exercised over 150+ temporary objects). Also confirms `&mut self` mutators are intentionally *not* bound (see below).
- **`demo_ractor.rb`** — calls module functions from non-main Ractors (single and 16-way concurrent). Green only because the extension declares `rb_ext_ractor_safe(true)` (from `ractor_safe = true` in `boltffi.toml`); an unmarked extension raises `Ractor::UnsafeError` on the first call inside a Ractor.

## `&self` is the only receiver

A Ruby object is a GC-managed handle that's freely aliased and shared across
threads/Ractors, so neither owned `self` (Ruby owns the box) nor `&mut self`
(Rust's exclusive-borrow guarantee can't hold when the receiver is aliased —
including self-aliasing like `c.merge(c)`) can be bound soundly. The lowerer
skips `&mut self` methods and warns loudly, telling the author to switch to
`&self` + interior mutability (`RefCell`/`Mutex`) — the rb-sys/magnus pattern.
The rest of the class still generates.

`StateHolder`, `CounterSingleThreaded`, and `AccumulatorSingleThreaded` in
`rust/src/classes/` all have `&mut self` mutators; `demo_classes.rb` calls only
their `&self`/constructor methods and asserts the mutators are absent
(`respond_to?` is `false`).

## What's deferred

Async methods, streams, and the `FfiBuf` wire codec for composite marshaling
(passing records / `Vec` / `Option` / value-returning `Result` across the FFI
boundary) are out of scope in PR #518. The generator filters those out at
lowering rather than emitting glue that can't encode the value, so anything you
see here actually works.
